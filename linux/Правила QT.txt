Правила QT:

1. Любой класс, использующий сигналы/слоты должен быть объявлен в *.h файле, иначе moc не видит его.
2. class MyCls: public QObject{ void Foo(){ emit ...; } }  Нельзя вызывать Foo из другого потока. emit
   должен вызывать свой поток.
   emit должен вызываться потоком, который владеет объёктом.
3. Для использования умных указателей подходит только QSharedPointer!
4. Нельзя выделять память с помощью malloc под объекты Qt! Только new.

5. Порядок использования QVector
	- Элементы расположены последовательно
	- Для доступа использовать функцию operator[]()
	- Для немодифицированного доступа лучше использовать функцию at()
	- Функция data возвращает указатель на массив (C++) элементов вектора
6. Когда возникает ошибка "undefined reference to vtable for class", то надо сделать
	Build->Run qmake
7. В классе CMyWork: public IWork размещаю статически объект QTimer timer. Объявляю слот в CMyWork::OnTimer
   и подключаю его к сигналу QTimer::timeout. В результате, connect не проходит! 
   Заработало только когда объект timer создал в свободной памяти:
   QTimer *timer=new QTimer(this);
